## Deskripsi

Proyek ini adalah tantangan pertama dari kurikulum **Foundations** di [The Odin Project](https://www.theodinproject.com/). Proyek ini bertujuan untuk membangun situs web resep sederhana menggunakan HTML murni guna mempraktikkan pemahaman dasar tentang struktur web sebelum melangkah ke CSS.

## Fitur & Konten

Referensi resep (https://www.allrecipes.com/)

## Teknologi yang Digunakan

- **HTML5**: Menggunakan elemen semantik untuk struktur data.
- **Git**: Manajemen versi dengan pendekatan _atomic commits_.
- **GitHub Pages**: Untuk _hosting_ dan akses _live_.

## Pelajaran yang Diambil (The Hard Part)

Selama pengerjaan 3 hari ini, pelajaran terpenting saya bukan hanya tentang _tag_ HTML, melainkan tentang etika koding:

### 1. Struktur Semantik HTML

Saya belajar bahwa HTML bukan hanya soal tampilan, tapi soal structure.

### 2. Git & Atomic Commits

Ini adalah bagian yang paling menantang menerapkan:

- **Commit Messages & Atomic Commits**: (https://cbea.ms/git-commit).

### 3. Manajemen Path (Relatif)

- Memahami cara berpindah folder menggunakan `../` dan `./` untuk menghubungkan halaman-halaman resep dan aset gambar secara efisien.

- Implementasi pengunaan ordered list dan unordered list.

## Penutup

Proyek ini memberi saya pondasi yang kuat dalam memahami bagaimana sebuah halaman web disusun dan bagaimana seorang developer harus mendokumentasikan setiap langkah perubahannya melalui Git.

## Description

This project is the first challenge of the **Foundations** curriculum at [The Odin Project](https://www.theodinproject.com/). It aims to build a simple recipe website using pure HTML to practice fundamental understanding of web structure before moving on to CSS.

## Features & Content

Recipe references: [AllRecipes](https://www.allrecipes.com/)

## Technologies Used

- **HTML5**: Utilizing semantic elements for structured data.
- **Git**: Version control management with an _atomic commits_ approach.
- **GitHub Pages**: For hosting and live deployment.

## Lessons Learned (The Hard Part)

During this 3-day journey, the most important lessons weren't just about HTML tags, but about coding ethics and best practices:

### 1. Semantic HTML Structure

I learned that HTML is not just about the visual output, but about building a meaningful and solid document structure.

### 2. Git & Atomic Commits

The most challenging part was consistently implementing:

- **Commit Messages & Atomic Commits**: Following the standards at [cbea.ms/git-commit](https://cbea.ms/git-commit).

### 3. Path Management (Relative)

- Understanding how to navigate through folders using `../` and `./` to link recipe pages and image assets efficiently.
- Proper implementation of ordered and unordered lists.

## Closing

This project has given me a strong foundation in understanding how a web page is structured and how a developer should document every step of their progress through Git.
